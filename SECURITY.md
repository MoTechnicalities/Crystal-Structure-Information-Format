# Security Policy

## Reporting a Vulnerability

If you discover a security issue, please do not open a public issue immediately.

Please report privately to the maintainer with:
1. A clear description of the issue.
2. Steps to reproduce.
3. Potential impact.
4. Suggested mitigation if available.

## Response Expectations

Target process:
1. Initial acknowledgment within 7 days.
2. Triage and severity assessment.
3. Coordinated fix and disclosure plan when applicable.

## Scope Notes

Current public scope is documentation and selected scripts. Security reports should prioritize:
1. API misuse vectors if connected deployments are used.
2. Data leakage risks in provenance/audit exports.
3. Unsafe defaults in reproducibility or runtime scripts.
