# Contributing to CIE/CSIF

Thanks for contributing to CIE/CSIF.

## Scope of This Public Repository

This repository currently exposes documentation and selected replication scripts.

Please:
1. Keep claims in docs aligned to runnable artifacts in this repo.
2. Preserve auditability and falsifiability language.
3. Avoid introducing undocumented benchmark claims.

## Contribution Types

Accepted contributions include:
1. Reproducibility improvements.
2. Documentation clarity and correction.
3. Testable experiment extensions.
4. Benchmark protocol tooling and validation utilities.

## Pull Request Rules

1. Open an issue first for major changes.
2. Keep PRs focused and small when possible.
3. Include before/after rationale.
4. Include reproduction steps for behavioral changes.

## Documentation Rules

1. Separate mechanism claims from external-validity claims.
2. Include explicit limitations when adding new results.
3. Use consistent terminology for phase conflict and audit traces.

## Review Criteria

A change is ready when it is:
1. Reproducible.
2. Auditable.
3. Honest about limits.
4. Aligned with roadmap priorities.
